# 30-Day Editorial Execution Plan

## Week 1 — 清理与基线

- 部署本优化版；
- 核对首页、博客、sitemap、RSS、robots 和 canonical；
- 记录 8 篇核心文章的曝光、点击和收录基线；
- 不新增批量模板页。

## Week 2 — Codex 专题

目标不是一次写 20 篇，而是完成 3 篇有真实操作证据的文章：

1. Codex 是什么，与 ChatGPT 对话式使用的区别；
2. Windows 安装、登录、项目权限和常见报错；
3. Codex 与 Claude Code 的场景比较。

每篇至少包含实测步骤、截图或错误案例，并引用官方资料。

## Week 3 — 图像模型专题

优先完成：

1. 图像模型中文提示词实测；
2. 海报文字、产品图和局部编辑案例；
3. 真实输出限制、尺寸、隐私和版权风险。

## Week 4 — 第三方与模型更新

- 重新核验 ZEOGPT 实时套餐和规则；
- 更新而不是复制旧文章；
- 对 AI 模型版本类文章做“版本变化、适用场景、迁移建议”，避免只堆模型名称；
- 根据真实曝光决定是否扩写。

## 每周节奏

```text
周一至周五：最多 1 篇达到发布门槛的新文
周六：更新旧文、补证据和内链
周日：合并重复意图、检查失效链接、复核 sitemap
```

质量不足时，宁可更新旧文，也不要为了“日更”创建新 URL。

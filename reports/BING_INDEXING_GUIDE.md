# Bing 收录与索引执行方案

核验日期：2026-06-22

> 规范的技术配置、原创内容、内链与持续更新可以提升被发现、抓取和收录的概率，但不能保证 Bing 必定收录、展示指定标题或摘要、获得首页排名。排序取决于内容质量、页面体验、域名历史、外链、重复度、查询竞争与算法。

## 一、注册与验证 Bing Webmaster Tools

1. 使用 Microsoft 账号登录 Bing Webmaster Tools；
2. 选择手动添加网站，输入正式 HTTPS 域名，或从已验证的 Google Search Console 导入；
3. 选择一种验证方式：
   - 下载 Bing 提供的 XML 文件并上传到网站根目录；
   - 将验证 Meta 标签放入首页 `<head>`；
   - 添加 DNS CNAME 记录；
4. 验证后确认平台显示的协议、主机名与站点 canonical 一致。

不要把 `example.com` 占位构建提交到 Bing。

## 二、提交 sitemap.xml

正式构建后访问：

```text
https://your-domain.example/sitemap.xml
```

确认：

- HTTP 200；
- XML 可解析；
- URL 全部使用正式 HTTPS 域名；
- 没有 404、测试域名和旧域名；
- 每个 URL 只有一个首选版本；
- lastmod 与真实更新对应。

在 Webmaster Tools 的 Sitemaps 区域提交完整 sitemap URL。本站当前构建包含 16 个首选页面。

## 三、URL Submission

对首页、博客首页和首批重点文章使用 URL Submission。新发布或实质更新后可以再次提交，但不要把反复提交当作排名手段。

优先顺序：

1. 首页；
2. `/blog/`；
3. 官网入口文章；
4. 中文版入口文章；
5. 官网打不开文章；
6. Plus 支付文章；
7. 镜像安全文章；
8. ZEOGPT 三篇文章；
9. 专题页与 FAQ。

大量自动提交可使用 Bing URL Submission API，需从 Webmaster Tools 获取 API Key，并遵守接口限制。

## 四、URL Inspection

对重点 URL 检查：

- Bing 是否能发现 URL；
- 最近抓取时间；
- 抓取是否成功；
- robots 是否允许；
- canonical 是否与预期一致；
- 页面是否已索引；
- 是否发现 SEO 或结构问题。

若页面已抓取未收录，不应只重复提交。先评估内容是否重复、过薄、商业比例过高、与已有文章意图重叠，或域名仍处于信任积累阶段。

## 五、robots.txt

正式域名构建应返回：

```text
User-agent: *
Allow: /

Sitemap: https://your-domain.example/sitemap.xml
```

测试：

- `/robots.txt` 可访问；
- 没有全站 Disallow；
- sitemap 使用正确域名；
- CDN 没有缓存占位阶段的禁止版本；
- 重要 CSS、JS 和图片未被屏蔽。

本项目在域名未填写时故意输出 `Disallow: /`，防止错误测试站被收录。

## 六、sitemap 格式检查

- XML 声明与 `<urlset>` 命名空间正确；
- 每个 `<loc>` 是绝对 HTTPS URL；
- URL 返回 200；
- 不包含 noindex 页面；
- 不包含重定向、404、查询参数或重复变体；
- lastmod 使用 YYYY-MM-DD；
- 页面删除后及时移出 sitemap；
- 超过 50,000 URL 或 50MB 时拆分索引。

## 七、文章页 SEO 检查清单

- 唯一 title，核心搜索意图靠前；
- 唯一 meta description；
- 一个清楚 H1；
- H2/H3 结构自然；
- canonical 指向自身正式 URL；
- 页面可在无 JavaScript时获得主要正文；
- 更新时间真实；
- 作者、分类、标签可见；
- Article、FAQ、Breadcrumb JSON-LD 与可见内容一致；
- 至少 3 条有价值内链；
- 外部权威链接指向一手来源；
- 图片有可读 alt；
- 移动端不横向溢出；
- CTA 不遮挡正文；
- 非官方与披露清楚；
- 不使用无法证实的绝对承诺。

## 八、内链策略

### 首页

链接到博客、五个核心主题、ZEOGPT、套餐、FAQ 和热门文章。核心文章应在两次点击内到达。

### 博客页

提供推荐、最新、分类和关键词入口。列表初始 HTML 必须包含文章链接，不只依赖客户端筛选。

### 文章页

链接到上位主题页、相邻问题、对比文章和安全文章。官方入口文章与镜像安全文章作为多个内容集群的枢纽。

避免：

- 所有链接使用完全相同锚文本；
- 每篇文章都链接全部页面；
- 只链商业页不链教程；
- 新文章没有任何入口；
- 依赖 footer 堆积几十个关键词链接。

## 九、搜索结果标题优化

- 先回答查询，再加站名；
- 保持自然，避免连续重复同一关键词；
- title 与 H1 可以略有差异但意图一致；
- 不虚构年份、价格、官方身份或“最新”；
- 标题被 Bing 改写是正常现象，不能保证展示提交的 title；
- 若标题频繁被改写，检查是否过长、堆词或与正文不一致。

## 十、摘要优化

Meta description 用 1–2 句回答“页面解决什么、有什么边界”。正文首段也应快速给结论，因为搜索引擎可能从正文抽取摘要。

不要把摘要写成纯广告或塞满同义关键词。对第三方页面加入“第三方”“公开页面快照”“以实时页面为准”等信任信息。

## 十一、观察排名与流量

在 Bing Webmaster Tools 查看：

- Impressions；
- Clicks；
- Average position；
- Crawl requests；
- Indexed pages；
- Search keywords；
- Page-level performance。

用品牌词、问题词和商业词分组观察，不以单日波动下结论。新站按 28 天与 90 天窗口评估。

## 十二、已抓取但未收录的判断

可能原因：

- 与站内或外部页面高度重复；
- 内容过薄或主要是；
- 页面与搜索意图不匹配；
- canonical 指向其他 URL；
- noindex 或 robots 冲突；
- 大量低质量页面稀释抓取；
- 域名很新，信任信号不足；
- 服务器不稳定或返回软 404；
- 页面更新时间与内容不符。

处理方式是改善页面价值、合并重复意图、加强内链和来源，而不是每天重复提交。

## 十三、减少重复内容

- 一个主题只设一个主要 URL；
- 统一 HTTPS、www 选择和尾斜杠；
- 旧 URL 使用 301，不同时保留近似正文；
- title、description、H1 和首段保持唯一；
- 文章不批量替换词语生成；
- 套餐快照与品牌介绍页承担不同意图；
- 标签和分类页只有在有独立价值时才开放索引。

## 十四、提升质量信号

- 以一手来源为事实基线；
- 显示核验日期与实质更新时间；
- 对不确定信息使用条件化表达；
- 提供清单、对比表、失败场景和风险边界；
- 删除过时截图与失效链接；
- 保持清楚作者和站点声明；
- 建立相关站点的自然引用和品牌提及，不购买垃圾外链；
- 页面性能、移动可读性和稳定 HTTPS 达标。

## 十五、新站冷启动发布节奏

建议首周上线完整基础页面和 8 篇重点文章，之后：

- 第 2–4 周：每周 2–3 篇；
- 第 2–3 个月：每周 2 篇，并更新已有高曝光页面；
- 每月：核验价格、模型、支付和支持地区；
- 每季度：做内容合并、失效链接和关键词重叠检查。

不要一次发布几十篇模板化短文。稳定、实质更新比表面数量更重要。

## 十六、IndexNow

Bing 支持 IndexNow 协议。可在站点根目录放置平台生成的 key 文件，并在发布或实质更新时提交变更 URL。IndexNow 用于通知搜索引擎 URL 变化，不等于保证抓取、收录或排名。

- [ ] robots 为 Allow；

- [ ] Bing 验证完成；
- [ ] sitemap 已提交；
- [ ] 重点 URL 已检查并提交；
- [ ] 首页到重点文章不超过两次点击；

# 参考站结构分析

参考：`https://www.chatgpt-chinese.com/blog/`  
核验日期：2026-06-22

## 原则

本项目只参考信息架构、栏目组织和 SEO 页面逻辑，不复制参考站代码、Logo、品牌、配色细节、版式尺寸或文章原文。新站的组件、CSS、文案、文章和数据结构均重新编写。

## 博客首页

参考站使用清晰的教程中心结构：顶部站点导航、左侧分类、中央文章列表和辅助信息。文章入口按主题集中，用户可以从 ChatGPT、Claude、Gemini 等栏目进入。

可借鉴：

- 列表页集中承接大量搜索意图；
- 推荐内容与最新内容并列；
- 分类和关键词作为内链入口；
- 标题直接回答用户问题；
- 文章卡片包含摘要和更新时间。

不能照搬：

- 相同品牌名称、Logo 和视觉细节；
- 相同标题句式的大规模复制；
- 原文段落、图片、CSS 和组件实现；
- 过度相似的栏目顺序、卡片尺寸与布局细节。

## 文章详情页

典型详情页具有：

- 唯一 H1；
- 更新时间；
- 分类和标签；
- 左侧栏目导航；
- 右侧目录；
- 多级 H2/H3；
- 常见问题；
- 相关阅读；
- 前后文章；
- 静态可抓取正文。

这些是常见内容站模式，不是某一站独占的设计。新项目采用 VitePress 默认主题能力与原创组件重新实现。

## URL 与静态页面

参考站使用可读的静态路径和明确的文章层级，搜索引擎无需执行复杂应用逻辑即可获得正文。新项目统一使用英文目录 slug，例如：

```text
/blog/chatgpt-official-entry-domestic-use/
```

## 移动端

参考站的导航、正文、目录和列表能在窄屏重新排列。新项目通过自有响应式 CSS，在 960px 与 640px 断点调整卡片、侧栏、CTA 和表格。

## SEO title 与 description 风格

可借鉴“核心查询 + 具体问题 + 年份或指南”的信息密度，但不机械堆叠关键词。新项目为每页写唯一 title 与 description，并把“非官方”和风险边界放入正文与全站声明。

## 为什么有利于抓取

- 页面是静态 HTML；
- 文章入口由首页、博客、侧栏和相关文章多次链接；
- URL 层级稳定；
- H1/H2/H3 清楚；
- 更新时间与主题可见；
- 内容不是只有图片或客户端异步数据；
- 主题聚类形成内部语义网络。

这些条件可改善搜索引擎发现和理解，但不能单独保证收录或排名。

## 新项目的差异化

- 站名、配色、图标和首页设计原创；
- 更明确地区分 OpenAI 官方与第三方服务；
- 对 ZEOGPT 使用披露和风险说明；
- 域名占位时自动 noindex，防止错误 canonical；
- 增加 Article / FAQ / Breadcrumb JSON-LD；
- 增加自动审计和链接检查；
- 60 篇矩阵按意图、CTA 适配和更新频率管理。

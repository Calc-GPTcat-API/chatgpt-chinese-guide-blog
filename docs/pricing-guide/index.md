# ZEOGPT 套餐核验与选择方法

第三方平台的价格、模型和额度会变化。与其把过期价格表长期留在文章里，更稳妥的做法是用统一清单检查实时套餐。

## 付款前比较表

| 项目 | 要问的问题 |
| --- | --- |
| 计费周期 | 月付、季付还是其他周期？是否自动续费？ |
| 额度 | 按消息、Token、倍率还是公平使用规则计算？ |
| 模型 | 目标模型是否当前可用？是否有单独限额？ |
| 并发与速度 | 高峰期、长对话、文件上传或代码任务有何限制？ |
| 退款 | 哪些情况可退款？已经使用额度后如何处理？ |
| 账号 | 是否为独立账号？如何找回？能否修改邮箱和密码？ |
| 数据 | 对话是否保存、如何删除、由谁处理？ |
| 售后 | 公开客服渠道、响应时间和争议处理规则是什么？ |

## 选择顺序

1. 先写下自己的主要任务，例如普通问答、长文、图片、代码或多模型比较；
2. 再查看实时套餐是否明确支持该任务；
3. 用最低可满足需求的套餐开始，不为不确定的未来需求预付过高成本；
4. 截图保存购买时的价格、权益、退款和续费说明；
5. 不把“无限”“长期稳定”等营销词当作无条件承诺。
